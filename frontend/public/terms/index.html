数据使用说明

最近更新：2026 年 7 月 19 日

1. 应用概述

Tova Ads 是一个 Facebook 广告管理平台（SaaS），帮助广告主通过自动化规则监控广告表现、执行止损策略、管理落地页和追踪转化。

2. 申请的 Facebook 权限及用途

权限用途是否必需
ads_management管理广告：暂停/恢复广告、修改预算、更改状态（自动止损核心功能）必需
ads_read读取广告结构、性能数据（消耗、转化、覆盖等）用于看板和巡检必需
pages_show_list显示用户管理的主页列表（合规认证、主页选择）按需
pages_read_engagement读取主页互动数据按需
email获取用户邮箱（账户标识、通知）必需

3. 数据使用方式

  • 广告监控：定时（每 5-30 分钟可配）从 Facebook 拉取广告性能快照，用于趋势分析和止损规则评估。
  • 自动止损：根据用户配置的规则（如消耗≥$20 无转化），自动暂停表现不佳的广告。暂停操作通过 Facebook Marketing API 执行。
  • 落地页管理：生成和管理落地页（部署到 Cloudflare Pages），配置流量防护（国家/设备/VPN 屏蔽）。
  • 转化追踪：在落地页上部署 Facebook Pixel，追踪访问和转化事件，回传给 Facebook 用于广告归因。
  • 告警通知：通过站内消息和 Telegram 推送广告性能、账户状态、止损触发等告警。

4. 数据不用于

  • 我们不将您的广告数据出售给第三方。
  • 我们不将您的数据用于定向广告。
  • 我们不与外部共享您的 Facebook 访问令牌。
  • 我们不在本服务之外使用您的落地页访客数据。

5. 数据删除

当您撤销 Facebook 授权或取消纳管账户时：

  • 立即停止获取新的广告数据。
  • 清除该账户的广告配置缓存。
  • 历史性能数据按保留策略（默认 90 天）自动过期删除。
  • 您可联系 support@tovaads.com 要求立即删除所有关联数据。

6. 用户责任

  • 您需确保拥有管理所绑定广告账户的合法权限。
  • 您需遵守 Facebook 广告政策和社区标准。
  • 您需对自动止损规则的配置负责（系统按您设的规则执行）。
  • 您不得利用本服务进行欺诈、刷量或违反 Facebook 政策的行为。

7. 服务限制

  • 本服务依赖 Facebook API 可用性。Facebook API 故障可能导致数据延迟或操作失败。
  • 自动止损基于预设规则执行，不保证广告效果，用户应定期审查规则配置。
  • 落地页防护基于 IP/UA/ASN 等特征，无法 100% 阻止所有无效流量。
